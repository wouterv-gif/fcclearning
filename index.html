Lesson Notes: Semantic HTML 2025-10-21

In this lesson I started looking at semantic HTML. I learned about different elements such as <i>, <em>, <b>, and <strong>, and how they differ. The first two mainly add visual or tonal emphasis, while the latter two give semantic meaning — helping browsers and screen readers understand importance or stress in content.

Then I moved on to description lists, which are used for things like glossaries or definitions. A description list uses a <dl> container, with one or more <dt> (description terms) followed by <dd> (description details). A single term can even have multiple details attached to it. Example:

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
</dl>

Next came block and inline quotes. The <blockquote> element is used for longer quotations, and it can include a cite attribute that holds a source URL — useful for screen readers and crawlers. The <q> element, on the other hand, is for short inline quotes that stay within a paragraph. There’s also a separate <cite> element, which is used to reference the name of a work or author, not a URL.

After that, I looked at abbreviations. Using <abbr> alone doesn’t change anything visually, but when combined with the title attribute — for example <abbr title="HyperText Markup Language">HTML</abbr> — the full term appears when hovering over it. That tiny touch of accessibility is pretty neat.

Then came the address element. It’s surprisingly versatile — perfect for contact sections, author info, or business pages. It should be used instead of a generic <div> when marking up contact details. Inside it, I can use <br> for line breaks, and links like tel: or mailto: to make phone numbers and emails clickable:

<address>
  John Doe<br>
  Streetname 123<br>
  Utrecht, Netherlands<br>
  <a href="tel:+31612345678">+31 6 12345678</a><br>
  <a href="mailto:info@example.com">info@example.com</a>
</address>

I wrapped up with the time element, which marks moments or dates. The datetime attribute converts it into a machine-readable format using the ISO 8601 standard. Example:

<time datetime="2025-10-21T19:48">This is right now.</time>

Overall, this lesson really showed how much clarity and meaning semantic HTML adds — not just for styling, but for accessibility, SEO, and general structure. It’s the difference between a web page that only looks right, and one that actually communicates right.

Lesson Notes: More on Semantic HTML 2025-10-30

More on Semantic HTML

Today we continued with semantic HTML, looking at a few inline elements that add meaning or structure to text.

We looked at the <sup> element, which is used to display text slightly above the normal line. For example, writing 4<sup>2</sup> will render as 42. The opposite, <sub>, displays text below the line — for instance, CO<sub>2</sub> will show as CO2.

The <code> element was also discussed — it's used for inline code snippets, such as demonstrating syntax or showing a command. For example: color: red;.

We then looked at <s> (for strikethrough text, often used to show something no longer relevant), and <u> (for unarticulated annotation — typically underlined text).

Finally, the <ruby> element was introduced. This one's mainly used for East Asian languages to show pronunciation above the characters. Inside it, the <rp> tag (ruby parenthesis) can be used to add fallback parentheses in case ruby annotations aren't supported.

Semantic HTML - Review 2025-10-30

Semantic HTML — Review

Semantic HTML is important to maintain the structural hierarchy of content. For example, using the heading elements <h1> through <h6> in the correct order.

Presentational HTML elements such as <center>, <big>, and <font> are considered outdated. Semantic elements, on the other hand, add meaning and structure to content.

Common semantic elements include <nav>, <header>, and <figure>. The <header> element defines the header of a document or section, while the <main> element contains the main content of a web page. The <section> element divides content into smaller, logical parts. The <nav> element represents navigation links, and <figure> is used to group illustrations or diagrams.

<em> (emphasis) marks text with stress emphasis, while <i> (idiomatic text) highlights alternative voice or mood, foreign or technical terms, or thoughts. The lang attribute can specify the language of content inside an element. The <strong> element indicates importance, whereas <b> (bring attention to) highlights text without adding importance—useful for things like product names in reviews.

The <dl> (description list), <dt> (description term), and <dd> (description detail) elements define term–description pairs. The <blockquote> element represents a quoted section from another source, with the cite attribute specifying the source URL. The <cite> element itself is used to visually attribute a referenced work. Short inline quotations use the <q> element.

The <abbr> element represents abbreviations or acronyms. With its title attribute, you can define the full form (e.g. HTML).

<address> defines contact information, while <time> represents a time or date. The datetime attribute makes it machine-readable (ISO 8601 format).

For mathematical or chemical notation, <sup> (superscript) and <sub> (subscript) are used—for example 42 and CO2.

The <code> element represents inline code fragments, and when combined with <pre>, it can show preformatted blocks:

body {
  color: red;
}

<u> (unarticulated annotation) marks text with non-textual annotation, and <ruby> is used for East Asian typography to show pronunciation or meaning. Inside <ruby>, the <rt> (ruby text) provides the annotation itself, and <rp> (ruby parenthesis) defines fallback parentheses for unsupported browsers.

Finally, the <s> (strikethrough) element represents content that is no longer accurate or relevant.

That wraps up the review of semantic HTML elements.

Introduction to Forms 2025-10-30

HTML Forms — Introduction

Form elements are used to collect information from the user. A form is created using the <form> element, which acts as a container for input fields, labels, buttons, and other form-related elements.

The action attribute of the form specifies where the data will be sent when the form is submitted (usually a server or script). In this example, the action attribute is left empty (action=""), which means the form doesn't actually send data anywhere yet — perfect for practice or static pages. 

<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" placeholder="example@email.com" />
</form>

Inside the form we use a <label> element to describe what kind of input the user should provide. The for attribute on the label connects it to an input field with a matching id. This connection improves accessibility and lets users click the label to focus the corresponding input.

The <input> element is used to actually collect data. Here, the type="email" attribute specifies that the input expects an email address — browsers can then provide email-specific validation and a suitable keyboard on mobile devices. The placeholder attribute shows example text (like example@email.com) to guide the user, which disappears when the user starts typing.

Later on, forms can include various types of inputs — such as text, password, number, checkbox, radio, and more — as well as elements like <textarea> for longer text and <button> for submission.
